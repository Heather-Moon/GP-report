## 2. 화면 구조 (IA)

```
대시보드
├── 메인 페이지 (평가 개요)
│   ├── 평가 개요 바 (응시자 현황, 평균 스킬 확보율)
│   ├── [탭] 평가 결과
│   │   ├── 점수 분포 (KDE 곡선, 과목별 탭)
│   │   ├── 역량 획득 현황 (도넛 차트)
│   │   └── 평가 총평 (전체 + 트랙별)
│   └── [탭] 응시자 목록
│       └── 필터/정렬/검색 + 테이블 + 페이지네이션
│
└── 평가자 개인 페이지
    ├── 헤더: 이름 / 이메일 / 완료 여부 / 날짜 / 소요시간 / 행동 분석 버튼
    │   └── 행동 분석 패널: 영상 | 타임라인
    ├── [탭] 결과 요약
    │   ├── 총점 / 획득 역량 수 / 순위
    │   ├── 과목별 점수 (Progress Bar)
    │   ├── 점수 분포 (KDE, 내 위치 마커)
    │   └── 시험 결과 총평
    ├── [탭] 클라우드 & 분산 시스템
    ├── [탭] 클라우드 컴퓨팅
    └── [탭] Python
        ※ 트랙 탭 공통 구성:
           - 트랙 설명
           - 트랙 점수 / 등수 / 소요시간 / 획득 역량 수(확보율)
           - 점수 분포 (KDE, 내 위치 마커)
           - 역량별 레이더 (나 vs 전체 평균, 호버 툴팁)
           - 평가 총평
           - 스킬별 세부 결과 (항상 펼침, 장단점 포함)
```

## 3. 사용자 플로우

```
[메인 페이지 진입]
    │
    ├── [평가 결과 탭] 통계/차트 확인
    │
    └── [응시자 목록 탭]
            │
            ├── 필터: 응시 완료 / 미완료
            ├── 정렬: 점수 / 스킬확보율 / 소요시간
            ├── 검색: 이름, 이메일
            │
            └── "결과 보기" 버튼 클릭
                    │
                [평가자 개인 페이지]
                    ├── 헤더 확인 (이메일/완료여부/날짜/소요시간)
                    ├── 행동 분석 버튼 → 영상 / 타임라인
                    │
                    ├── [결과 요약 탭]
                    │   ├── 총점 / 획득 역량 / 순위 카드
                    │   ├── 과목별 점수 바
                    │   ├── 점수 분포 (내 위치 표시)
                    │   └── 시험 결과 총평
                    │
                    └── [트랙별 탭 (3개)]
                        ├── 트랙 설명
                        ├── 점수 / 등수 / 소요시간 / 역량 카드
                        ├── 점수 분포 (내 위치 표시)
                        ├── 레이더 차트 (나 짙은색 / 평균 옅은색, 호버)
                        ├── 평가 총평
                        └── 스킬별 세부 결과 (펼침, 강점/단점)
```

## 4. 화면 상세 기획

### 4.1 메인 페이지 (평가 개요 대시보드)

**페이지 목적:** 전체 평가 현황을 한눈에 파악하고 평가자 목록을 관리

---

#### 4.1.1 평가 개요 바

| 구성 요소 | 내용 |
|----------|------|
| 평가 기간 | 시작일 ~ 종료일, 제한 시간 |
| 응시자 현황 | 완료 N명 (X%) / 전체 N명 |
| 평균 스킬 확보율 | X% |

---

#### 4.1.2 평가 결과 탭

**점수 분포**

| 구성 요소 | 내용 |
|----------|------|
| 탭 버튼 | 전체 평가 결과 / 분산 아키텍처 / 클라우드 컴퓨팅 / Python |
| KDE 곡선 | 매끄러운 밀도 곡선, 불합격(빨강)/합격(초록) 배경 구간 전체 폭 표시 |
| 합격선 | 80점 기준 초록 실선 |
| 평균선 | 주황 점선 |
| 범례 | 그래프 하단에 평균(주황 점선) / 합격선(초록 실선) 범례 표시 |
| 우측 통계 | 평균 / 중앙값 / 범위 / 표준편차 (레이블 → 값 순) |

**역량 획득 현황**

| 구성 요소 | 내용 |
|----------|------|
| 요약 통계 | 전체 획득률 / 인당 평균 획득 / 전체 스킬 수 |
| 도넛 차트 | 전체 + 트랙별 (전체와 트랙 사이 구분선) |

**평가 총평**

| 구성 요소 | 내용 |
|----------|------|
| 전체 총평 | 평균·분포·역량 획득 종합 텍스트 |
| 트랙별 총평 | 각 트랙 이름 + 텍스트 (구분선으로 분리) |

---

#### 4.1.3 응시자 목록 탭

| 컨트롤 | 옵션 |
|--------|------|
| 검색 | 이름, 이메일 |
| 상태 필터 | 전체 / 완료 / 미완료 |
| 정렬 | 점수순 / 스킬확보율순 / 소요시간순 |

**테이블 컬럼:** 응시자 / 상태 / 총점 / 스킬 확보율 / 소요시간 / 결과 보기 버튼

---

### 4.2 평가자 개인 페이지

**페이지 목적:** 특정 평가자의 전체 시험 결과를 상세하게 파악

**진입 경로:** 응시자 목록 탭 > "결과 보기" 버튼 클릭

---

#### 4.2.1 페이지 헤더

| 컴포넌트 | 내용 |
|----------|------|
| 아바타 | 이름 첫 글자 |
| 이름 | 대형 타이틀 |
| 메타 정보 | ✉ 이메일 / 완료 여부 배지 / 📅 날짜 / ⏱ 소요시간 |
| 행동 분석 버튼 | 클릭 시 모달 팝업 |
| 행동 분석 모달 | 왼쪽: 화면 녹화 영상 + 재생 컨트롤 / 오른쪽: 타임라인 (이벤트 목록) |

---

#### 4.2.2 결과 요약 탭

**블록 1 — 평가 결과**

| 구성 요소 | 내용 |
|----------|------|
| 지표 카드 3열 | 총점 (합격 여부) / 순위 / 소요시간 (전체 제한 2:40:00 기준) |
| 트랙별 점수 3열 | 트랙명 + 점수 |

**블록 2 — 점수 분포 & 획득 역량**

| 구성 요소 | 내용 |
|----------|------|
| KDE 곡선 | 내 위치 마커 (콜아웃 박스, 점수 표시), 합격선/평균선 색상 구분 |
| 우측 통계 패널 | 평균 / 중앙값 / 표준편차 |
| 범례 | 그래프 하단 평균(주황 점선) / 합격선(초록 실선) |
| 획득 역량 도넛 | 전체 + 트랙별 (3개) 도넛 차트 |

**시험 결과 총평**

| 구성 요소 | 내용 |
|----------|------|
| 총평 | 종합 텍스트 |

---

#### 4.2.3 트랙별 탭 (클라우드 & 분산 시스템 / 클라우드 컴퓨팅 / Python)

**트랙별 제한시간**

| 트랙 | 제한시간 |
|------|---------|
| 클라우드 & 분산 시스템 아키텍처 | 0:40:00 |
| 클라우드 컴퓨팅 | 0:30:00 |
| Python | 2:00:00 |

**블록 1 — 평가 결과**

| 구성 요소 | 내용 |
|----------|------|
| 지표 카드 4열 | 트랙 점수 / 순위 / 역량 확보 (N/전체개) / 소요시간 (트랙 제한시간 기준) |

**블록 2 — 점수 분포 & 역량별 분석**

| 구성 요소 | 내용 |
|----------|------|
| 점수 분포 | KDE 곡선, 내 위치 마커, 합격선/평균선 색상 구분, 범례, 우측 통계 패널 |
| 역량별 분석 | 나(짙은색 실선) vs 전체 평균(옅은색 점선) 레이더 차트, 축 호버 시 툴팁 |
| 역량 현황 목록 | 스킬명 + 획득/보완/미획득 상태 (레이더 하단) |

**평가 총평 / 스킬별 세부 결과**

| 구성 요소 | 내용 |
|----------|------|
| 평가 총평 | 트랙 총평 텍스트 |
| 스킬별 세부 결과 | 항상 펼침 — 배지 / 스킬명 / 전체 획득률 / 설명 / 강점·보완·단점 판정 |

**레이더 차트 상세**

| 항목 | 내용 |
|------|------|
| 나 (My) | 짙은색 실선 폴리곤 |
| 전체 평균 | 옅은 회색 점선 폴리곤 |
| 호버 | 축 근처 hover 시 "내 점수 X% \| 전체 평균 Y%" 툴팁 표시 |

**스킬 세부 판정 기준**

| 레벨 | 판정 | 배경색 |
|------|------|--------|
| 획득 | 강점 | 초록 |
| 보완 필요 | 보완 | 노랑 |
| 미획득 | 단점 | 빨강 |

## 변경 이력 — 2026.04.18

> 실제 응시자 데이터는 손덕인 평가자의 결과에서 확인하실 수 있습니다.

---

### 1. 소요시간 분포 그래프 추가

평가 결과 탭 내 점수 분포 아래에 소요시간 분포 그래프를 추가하였습니다.
전체 평가 및 트랙별(분산 아키텍처 / 클라우드 컴퓨팅 / Python) 탭으로 구분하여 확인할 수 있습니다.

---

### 2. 평가 개요 대시보드 점수 분포: KDE 곡선 → 히스토그램 변경

평가 개요 대시보드의 점수 분포 그래프를 KDE 곡선에서 히스토그램으로 변경하였습니다.

**변경 사유**

대시보드는 다수의 응시자를 한눈에 비교하는 공간으로, 히스토그램은 "몇 명이 몇 점대에 분포하는가"를 실제 수치 기반으로 직관적으로 전달합니다. KDE 곡선은 데이터를 수학적으로 평활화하는 과정에서 응시자 수가 적은 환경일수록 실제와 다른 분포 인상을 줄 수 있으며, 절댓값(인원 수)보다 밀도(상대적 높낮이)를 강조하는 특성상 전체 현황 파악보다는 분포 형태 분석에 더 적합합니다.

반면 평가자 개인 페이지에서는 "전체 집단 내 본인의 위치"를 연속적인 곡선 위에 마커로 표현하는 것이 상대적 위치 파악에 직관적이므로 KDE를 유지하였습니다.

---

### 3. 부정행위 지표 위치 변경 및 결과 등급 체계 개편

부정행위 지표를 응시자 목록 테이블의 별도 컬럼에서 평가자 개인 페이지 **결과 요약 블록**으로 이동하였습니다. 행동 분석 링크도 동일 위치에 배치하여 지표 확인 후 즉시 상세 분석으로 진입할 수 있도록 하였습니다.

결과 등급은 아래 기준으로 표시됩니다.(임시)

| 등급 | 기준 |
|------|------|
| 정상 | 위험 이벤트 0건 + 주의 이벤트 1건 이하 |
| 주의 | 위험 이벤트 1~2건 또는 주의 이벤트 2건 이상 |
| 위험 | 위험 이벤트 3건 이상 |

---

### 4. 평가자 개인 페이지 PDF 발급 버튼 추가

평가자 개인 페이지 헤더에 PDF 발급 버튼을 추가하였습니다.
확인 결과 현재 트랙별 결과 탭의 레이아웃이 PDF 출력 시 일부 수정이 필요한 상태로, 추가 보완이 필요합니다.

---

### 5. 트랙별 설명 노출 위치 조정

트랙별 설명은 평가를 설정한 기업 담당자 입장에서 반복 확인의 필요성이 낮다고 판단하여 **전체 결과 페이지(메인 대시보드)에서는 제외**하였습니다. 평가자 개인 페이지에서는 결과 요약 탭 내 **평가 구성** 섹션에 포함하여, 개인 결과를 해석하는 맥락 정보로 제공합니다.

---

### 6. 백분위 추가

백분위를 순위 카드 하단에 추가하였습니다 (예: 상위 15%).

**백분위 유지에 대한 논의**

**우리가 백분위를 통해 전달하려는 것**

절대 점수만으로는 파악하기 어려운 상대적 위치 정보를 제공합니다. 동일한 75점이라도 전체 상위 10%인지 하위 30%인지에 따라 채용·스킬업 판단의 방향이 달라질 수 있으며, 이를 직관적으로 전달하는 것이 목적입니다.

**고객(기업 담당자)이 백분위를 통해 기대하는 것**

기업 담당자는는 백분위를 통해 "이 응시자가 전체 집단 중 어느 수준인가"를 단번에 판단하고자 합니다. 합격/보류/불합격 기준을 절대 점수가 아닌 상위 N% 기준으로 유연하게 적용하거나, 상위 인원을 추려내는 데 활용할 수 있습니다. 다만 응시자 수가 적을수록 백분위의 변동폭이 커져 해석이 불안정해질 수 있으므로, 순위(N/전체)와 함께 제공함으로써 보완하는 것이 적절합니다.

**스킬 확보율 vs 역량 획득률 — 데이터 병행 제공 여부 논의**

현재 스킬 확보율(개인별 획득 역량 비율의 평균)과 역량 획득률(전체 집단 기준 역량별 획득 비율) 두 가지 지표를 모두 노출하고 있습니다. 두 지표가 측정하는 대상이 다르므로 각각의 필요성과 중복 여부에 대한 추가 논의가 필요합니다.

---

### 7. 주요 지표에 ? 아이콘 및 호버 툴팁 설명 추가

처음 접하는 평가자(or 기업 담당자)가 각 지표의 의미를 별도 안내 없이 파악할 수 있도록, 주요 항목 옆에 ? 아이콘을 추가하고 마우스를 올렸을 때 간단한 설명이 표시되도록 하였습니다.

| 위치 | 항목 | 툴팁 내용 요약 |
|------|------|--------------|
| 평가 개요 바 | 평균 스킬 확보율 | 완료 응시자 각각의 스킬 확보율을 평균낸 값. 개인별 스킬 확보율은 획득 역량 수 ÷ 전체 역량 수 |
| 평가 결과 탭 | 점수 분포 | 히스토그램 그래프 구성 요소 안내 (평균선·합격선 의미) |
| 평가 결과 탭 | 역량 획득 현황 | 도넛 차트 수치(인당 평균 획득 수 / 전체 역량 수) 해석 방법 |
| 개인 결과 요약 | 부정행위 지표 | 정상·주의·위험 판정 기준 (위험 이벤트 건수 기반) |
| 개인 결과 요약 | 점수 분포 | KDE 곡선 구성 요소 안내 (색상 점·점수 박스가 본인 위치) |
| 개인 결과 요약 | 획득 역량 | 도넛 차트 수치 해석 방법 |
| 트랙별 탭 | 점수 분포 | 해당 트랙 KDE 곡선 구성 요소 안내 |
| 트랙별 탭 | 역량별 분석 | 레이더 차트 읽는 법 (색상선 = 본인, 회색 점선 = 전체 평균) |

PDF 출력 시에는 ? 아이콘이 자동으로 숨겨집니다.

## 변경 이력 — 2026.04.22

---

### 1. PDF 출력 시 부정행위 탭 누락 수정

PDF 출력 시 개인 페이지의 부정행위(결과 요약) 탭이 인쇄 결과에 포함되지 않던 문제를 수정하였습니다. `style.removeProperty('display')` 호출만으로는 CSS 규칙에 의해 숨김 상태가 유지되던 것이 원인으로, `beforeprint` 핸들러에서 모든 탭의 `display`를 명시적으로 `block`으로 설정하도록 변경하였습니다.

---

### 2. 점수 분포 히스토그램에 KDE 확률 분포 곡선 추가

메인 대시보드 및 개인 페이지의 모든 히스토그램에 KDE(커널 밀도 추정) 확률 분포 곡선을 오버레이하였습니다. 막대그래프로 인원 수를 직관적으로 파악하면서 동시에 분포의 전체적인 형태를 부드러운 곡선으로 확인할 수 있습니다.

---

### 3. 히스토그램 합격/불합격 색상 구분 및 합격선 제거

히스토그램 막대의 색상을 점수 구간에 따라 합격(초록)/불합격(빨강)으로 구분하도록 변경하였습니다. 별도의 합격선(수직선) 없이도 합격 기준(80점)이 직관적으로 전달되므로 합격선은 제거하였습니다. 범례도 동일하게 업데이트하였습니다.

---

### 4. 소요시간 분포 시각화 개편 — 버블 차트 · 도넛 차트 · 산점도 추가

소요시간 분포 탭에 히스토그램 외 3가지 차트를 추가하였습니다.

| 차트 | 내용 |
|------|------|
| 스트립 플롯 (버블) | 응시자별 소요시간을 점으로 표시. Y축은 랜덤 지터(jitter)를 적용하여 점 간 겹침을 최소화 |
| 도넛 차트 | 소요시간을 구간별로 분류하여 비율 표시. 범례에 실제 시간 범위 표기 |
| 산점도 | X축 소요시간, Y축 점수. 합격/불합격을 색으로 구분하며 평균 소요시간 기준선 표시 |

소요시간 구간 레이블을 "빠름/보통/느림/초과" 같은 정성적 표현 대신 실제 분 단위 시간 범위(예: ~64분, 64~112분)로 표시하도록 변경하였습니다.

---

### 5. 산점도 합격선 제거

소요시간 산점도에서 합격선(수평선)을 제거하였습니다. 합격/불합격 여부는 점 색상으로 이미 구분되므로 선이 불필요하다는 판단에 따른 조치입니다.

---

### 6. 응시자 목록 — 트랙별 상세보기 기능 추가

응시자 목록에서 완료 상태 응시자의 이름 옆 **상세보기** 버튼을 클릭하면 해당 행 아래에 트랙별 결과 테이블이 펼쳐집니다. 다시 클릭하면 닫힙니다.

표시 항목: 트랙명 / 점수 / 합격 여부 / 스킬 확보율 / 획득 스킬 수 / 소요시간

---

### 7. 응시자 목록 — 이상행동 컬럼 제거

이상행동 컬럼을 응시자 목록 테이블에서 제거하였습니다. 이상행동 관련 정보는 개인 페이지 결과 요약에서 확인할 수 있으며, 목록 단계에서의 노출은 불필요하다는 판단에 따른 조치입니다.

---

### 8. 상세보기 버튼 위치 정렬 개선

이름 셀 내 상세보기 버튼을 `justify-content: space-between` 레이아웃을 적용하여 이름 길이와 무관하게 항상 셀의 오른쪽 끝에 고정되도록 개선하였습니다.
